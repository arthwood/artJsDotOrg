ArtJs is JavaScript framework of general purpose. It allows you to select or manipulate DOM elements, perform AJAX requests, provides load of helper methods for String, Date and other native classes, etc.

Currently it supports FF, Chrome, Safari and IE8.

Zip package contains one art.js file that loads appropriate library depending on browser it is running on. Default load location is set to "javascripts/" but you can change it by setting your own path like this:

<script type="text/javascript">
  window.ArtJs_path = '../javascripts/';
</script>
<script src="../javascripts/art.js" type="text/javascript"></script>

This is the order in which framework classes are loaded:

  • Initialize
  • utils/Log
  • events/Delegate
  • utils/MathUtils
  • utils/ObjectUtils
  • utils/ArrayUtils
  • utils/ClassUtils
  • utils/StringUtils
  • utils/DateUtils
  • utils/ElementUtils
  • utils/EventUtils
  • modules/Locator
  • events/DelegateCollection
  • events/CustomEvent
  • events/MouseController
  • math/Point
  • math/Rectangle
  • dom/ElementBuilder
  • dom/Selector
  • events/Clock
  • data/List
  • data/Queue
  • events/QueuedClock
  • net/Ajax
  • ui/containers/ModalBox
  • ui/DatePicker
  • ui/Flash
  • tween/Base
  • tween/Fade
  • tween/Reveal
  • Finalize

Some of the classes are optional (e.g. DatePicker), other are core classes on which other might depend (e.g. Delegate), but until there is no clean distinction between those two groups ("core" and "more") all framework classes are being loaded.

Just after library is loaded there are two methods that you may want to use: ArtJs.globalize() and ArtJs.doInjection()

ArtJs.globalize() makes all the classes globally accessible in window object, as well as shorthand utility methods like: $, $$, $P, $B, $D, $DC, $E.

ArtJs.doInjection() makes an extensions on native classes like Object, Array, Date, Element etc. so that you can use:

arr.first();

otherwise you are still able to call the method using:

ArtJs.ArrayUtils.first(arr);
