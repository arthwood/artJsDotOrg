Routing

Routing allows you to map url to specific action.

URL

In ArtJs everything that is after "#" in URL is treated as route path. Once the page is loaded or user navigates between URLs ArtJs will parse the URL and assume the format: 

/<controller>/<action> (with optional "!" after leading "/")

Controllers

By default ArtJs does not respond to navigation between URLs - it uses default controller with empty implementation {{code('artjs.Controller')}}. You can define you own default controller by extending {{code('artjs.Controller')}} and implementing any methods that you expect to occur in URL.
If no matching method can be found in controller then {{code('index')}} is used.
If no matching controller can be found then {{code('artjs.Router.defaultController')}} is used.

Every action receive {{code('request')}} argument, which is a generic object with two properties:

  • fragments - {{code('Array<String>')}} containing any extra parts of URL splitted by '/'
  • controllerId - {{code('String')}} optional, exists only if default controller is used. Contains name of controller that couldn't be found.
