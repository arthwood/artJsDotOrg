Templates

Templates allow you to reuse same html file many times and feeding it with different content.

Templates library

Templates are can be stored anywhere in your project but default location is "templates" folder at the root. You can specify the location by setting artjs.TemplateLibrary.config.PATH.

Loading templates

Templates are loaded automatically by the framework. You can specify the list of templates to load by setting artjs.TemplateLibrary.config.TEMPLATES. What frameword does is it hides body until all templates are loaded. That's because whole body is treated as a template and you may use placeholders there as well. It prevents from displaying them as a text before template is actually rendered. After that is fires artjs.onLibraryLoad event.

Creating templates

Template files have format of html with additional placeholders. Placeholders have format of: 

{{<expression>}}
Where expression can be:
  • current scope's property (templates are always render with a scope object)
  • template method (you can find available methods in reference)

Accessing templates

In order to get loaded template use artjs.TemplateLibrary.getTemplate(id) where id is path to your template (e.g. 'list/item')

Rendering templates

render

Once you have a template you can populate it with the content. The process is called rendering and can be performed using artjs.TemplateBase.render(template, scope) where template is raw template as a String and scope is an object that will be used to provide the content.

renderInto

With artjs.TemplateBase.renderInto(element, template, scope) you can render the template directly into an element. In Addition it will evaluate any scripts in the template and scan for any Components.

renderElement

artjs.TemplateBase.renderElement(element, scope) will render inner html of the element and update its content.

renderTemplateInto

artjs.TemplateBase.renderElement(element, templateId, scope) is the same as artjs.TemplateBase.renderInto but you can provide templateId instead of raw template.
