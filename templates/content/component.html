Component

Components allow you to create visual element with encapsulated functionality.

Definition

In order to define a component, assign to any HTML element css class "art".
However this wont do much. It will only allow the framework to recognize the fact that this element is an component. In order to define a behaviour just assign another css class that corresponds to js class defined in your code, e.g.: 

<div class="art app-components-MyComponent"></div>

Here, framework will search for a class app.components.MyComponent and instantiate it whenever that element appears in UI.
That class must extend artjs.Component (see class definition) and you must call super() in the constructor. Doing that will give you access to element property which keeps a reference to the main element (the one with assigned css class).

Lifecycle

Keep in mind that framework will remove any components that are not on stage every 2 seconds. You may change it through artjs.ComponentSweeper.INTERVAL setting. This is obviously for performance reasons.

Lookup

If you know that your component is going to be unique you may want to assign its element an id attribute.
Each element with id can be referenced using onLoad() hook. If component is already instantiated and you call: 

artjs.Component.onLoad('myId', delegate);

delegate will be called immediately and will receive reference to component instance.
Otherwise it will be called as soon as HTML element is found and component is instantiated.
You can always try to reference you component by id using: 

artjs.Component.find('myId');

This will return reference to component if it has been already instantiated and false otherwise.
